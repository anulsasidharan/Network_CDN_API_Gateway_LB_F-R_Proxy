Forward Proxy :3128

A forward proxy acts on behalf of clients. Your browser/app sends requests to the proxy, which forwards them to the destination. The server never sees your real IP directly.

How it works: Client → Forward Proxy → Target Server. Used for outbound traffic control, content filtering, and anonymization in corporate networks.

Client → Forward Proxy (:3128) → Load Balancer → Backends. Try: curl http://localhost:3128/proxy/load-balancer/api/info

Nginx Config Highlights

# nginx/forward-proxy/nginx.conf
# Route: /proxy/<host>/<path> → http://<host><path>
location ~ ^/proxy/([^/]+)(/.*)$ {
    proxy_pass http://$upstream_host$upstream_path;
    proxy_set_header Via "nginx-forward-proxy";
}
# Production: proxy_connect module for curl -x support

Load Balancer :8081

Distributes incoming requests across 3 backend instances using round-robin. Each click should hit a different server.

How it works: Client → Load Balancer → [backend-1, backend-2, backend-3]. Amazon ELB, NGINX upstream, HAProxy.

Nginx Config Highlights

# nginx/load-balancer/nginx.conf
resolver 127.0.0.11 valid=10s;          # Docker DNS
set $backend_upstream backend:3000;
proxy_pass http://$backend_upstream;    # runtime DNS → round-robin

Reverse Proxy :8082

Sits in front of backend servers. Clients talk to the proxy; the proxy talks to the backend. Backend identity is concealed.

How it works: Client → Reverse Proxy → Backend. Used for SSL termination, load balancing, caching, and security (WAF).

Nginx Config Highlights

# nginx/reverse-proxy/nginx.conf
proxy_pass http://backend-1:3000;
proxy_hide_header X-Powered-By;     # conceal backend
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

API Gateway :8083

Single entry point routing to microservices with API key authentication and rate limiting (10 req/min per IP).

How it works: Client → API Gateway → [users-service, products-service, orders-service]. Handles auth, routing, throttling, CORS.

Nginx Config Highlights

# nginx/api-gateway/nginx.conf
limit_req_zone $binary_remote_addr zone=api_limit:10m rate=10r/m;
map $http_x_api_key $api_key_valid { "demo-key-123" 1; }
location /api/users { proxy_pass http://users-service:3000/api/users; }

CDN (Content Delivery Network) :8084

Edge server caches static assets. First request = MISS (fetches from origin). Subsequent requests = HIT (served from cache).

How it works: Client → CDN Edge (cached) → Origin Server. CloudFront, Cloudflare, Akamai cache static content globally.

Nginx Config Highlights

# nginx/cdn/nginx.conf
proxy_cache_path /var/cache/nginx keys_zone=static_cache:10m;
proxy_cache static_cache;
proxy_cache_valid 200 10m;
add_header X-Cache-Status $upstream_cache_status;
